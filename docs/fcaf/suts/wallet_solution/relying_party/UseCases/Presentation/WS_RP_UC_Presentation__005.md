## Test Scenario
1. The Wallet engages with the Verifier
2. The Verifier sends a Authorization Request with a DCQL-query
3. Wallet can sucessfully display info

## Expected results
1. Wallet and Verifier can interact.
2. Wallet receives the request.
3. The wallet clearly displays the Verifier’s Name and Logo so the user knows who is asking for data. The purpose. The exact things being requested. A security warning if verifier metadata cannot be verified.
